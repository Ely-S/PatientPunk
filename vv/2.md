## Gaps and Risks
- No static artifact proves the actual `MIN(post_date)` and `MAX(post_date)` of included reports for any drug.
- Paxlovid, colchicine, and naltrexone lack explicit analysis-time cutoff filters; correctness depends on database filenames and README-stated windows matching actual database contents.
- Naltrexone is the highest-risk query path because `_nalt_recs()` does not join `posts`, cannot inspect `post_date`, and deduplicates across two databases without a date predicate.
- The code does not enforce lower-bound window starts in analysis queries; start dates are implicit in the selected database slices.
- There is no static/manual evidence that posts near cutoffs were reviewed for discussion of the paired trial result.
- Static analysis cannot prove that Arctic Shift downloads themselves were made with the intended date ranges or that all `post_date` values are non-null, correctly normalized, and within cutoff.

## Suggested Follow-up Checks
- Run per-drug included-report window audits before dedup:
  ```sql
  SELECT lower(t.canonical_name) AS drug, MIN(p.post_date), MAX(p.post_date), COUNT(*)
  FROM treatment_reports tr
  JOIN treatment t ON tr.drug_id = t.id
  JOIN posts p ON tr.post_id = p.post_id
  WHERE lower(t.canonical_name) = 'famotidine'
    AND p.post_date < 1633046400;
  ```
- Repeat the same audit for loratadine and prednisone with `p.post_date < 1633046400`; verify max dates are before `2021-10-01` UTC and min dates are on/after `2021-05-01` UTC.
- For paxlovid, run the same join without relying on the filename and verify `MIN(post_date) >= 1709251200` and `MAX(post_date) < 1717718400` (2024-03-01 through 2024-06-06 UTC, exclusive of 2024-06-07 if using whole-day cutoff).
- For colchicine, verify `MIN(post_date) >= 1609459200` and `MAX(post_date) < 1640995200` for the included reports.
- For naltrexone, audit both source databases with a `posts` join and aliases `('naltrexone', 'low dose naltrexone', 'ldn')`; verify combined included reports are `>= 1609459200` and `< 1643587200` (through 2022-01-30 UTC if exclusive of 2022-01-31).
- Add an analysis-time assertion query that fails the build if any included report falls outside its V2 window or has `post_date IS NULL`.
- Sample cutoff-adjacent included text, for example the latest 25 reports per drug, and search/review `body_text` and `title` for paired-trial terms: `Glynne`, `STOP-PASC`, `Bassi`, `O'Kelly`, `Utrero`, `trial`, `RCT`, `publication`, `published`, `study result`.
