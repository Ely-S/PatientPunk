# V1. Data Provenance and Freeze

## Verdict
Partial. The reproducibility package names and downloads the five processed databases used by the final build, but static evidence does not establish immutable raw/processed file hashes, exact Arctic Shift acquisition configuration, scrape dates, or final figure/table binding to a specific code revision and run metadata.

- `docs/RCT_historical_validation/VV_PLAN.md:35-54` defines V1 checks: Arctic Shift source/configuration, file names/sizes/SHA-256 hashes, five analysis databases, and final metadata including git commit, model names, prompt versions, run IDs, and database hashes.
- `docs/RCT_historical_validation/README.md:20-30` documents download URLs and approximate sizes for the five processed SQLite databases; `README.md:32-39` documents raw JSON download URLs for four raw Reddit JSON files, but not raw sizes or hashes.
- `docs/RCT_historical_validation/README.md:73-85` claims the databases are full, unmodified SQLite copies scraped from `r/covidlonghaulers` and describes the expected schema; `README.md:77-83` documents database date windows and drug coverage.
- `docs/RCT_historical_validation/_build_paper_figures.py:51-61` opens exactly the five V1 databases from `data/`; `_build_paper_figures.py:327-330` builds and exports `output/paper_figures*` and `figure1.png`.
- `docs/RCT_historical_validation/_build_paper_figures.py:107-161` derives final per-drug rows from those database connections, with naltrexone combined from `colchicine_naltrexone_year_2021.db` and `naltrexone_jan_2022.db`.
- `docs/RCT_historical_validation/_gen_analysis.py:29-40` generates `analysis.ipynb` with the same five database paths; `_gen_analysis.py:92-133` builds the primary response data from those connections.
- `docs/RCT_historical_validation/build_notebook.py:126-163` writes, executes, and exports notebooks, but only injects a relative DB path and does not record input hashes, git commit, run IDs, or model/prompt metadata.
- `schema.sql:43-49` defines `extraction_runs` with `run_id`, `run_at`, `commit_hash`, `extraction_type`, and JSON `config`; `schema.sql:80-89` links `treatment_reports` to `run_id`.
- `src/run_sentiment_pipeline.py:55-66` records model names and a git commit hash into `extraction_runs` via `ReportWriter`; `src/utilities/db.py:68-79` persists that metadata and assigns `run_id`.
- `src/utilities/__init__.py:94-108` resolves model names and current git commit, returning `"unknown"` if git metadata is unavailable.
- `src/import_posts.py:51-55` imports JSON into SQLite and sets `scraped_at` to import time; `src/import_posts.py:86-100` inserts users/posts and cleans dangling parent IDs.
- `Scrapers/README.md:72-79` and `Scrapers/SCRAPER_HELP.md:116-119` document Arctic Shift as the scraper source and describe run metadata outputs for the scraper.
- `Scrapers/scrape_corpus.py:18-33` identifies Arctic Shift docs/base URL and fixed subreddit `covidlonghaulers`; `scrape_corpus.py:553-569` writes `corpus_metadata.json` with subreddit, window, scrape start/finish timestamps, and source URL.
- `Scrapers/transform_arctic_shift.py:3-18` documents conversion from Arctic Shift bulk post/comment NDJSON or `.zst` files; `transform_arctic_shift.py:121-125` accepts posts, comments, output path, and subreddit; `transform_arctic_shift.py:200-208` prints transformed output counts and date range.

## Gaps and Risks
- No static evidence found for SHA-256 hashes of the raw JSON files or processed SQLite databases, despite V1 requiring file names, sizes, and SHA-256 hashes.
- Raw JSON URLs are documented, but raw file sizes are not; processed database sizes are approximate MB values and are not cryptographic freeze evidence.
- The README documents processed DB and raw JSON download locations, but not exact Arctic Shift download commands, API parameters, bulk-download URLs, scrape dates, or per-dataset configuration used for the four historical raw JSON files.
- `scrape_corpus.py` can write scrape metadata, but V1 cannot be proven for these final datasets because the referenced package does not include or cite the corresponding `corpus_metadata.json` files.
- `transform_arctic_shift.py` converts bulk Arctic Shift files but does not write a provenance manifest containing source filenames, Arctic Shift URLs, date bounds, command line, scrape date, sizes, or hashes.
- Final figure/table generation does not record git commit, database hashes, extraction run IDs, model names, prompt versions, or the exact `extraction_runs` rows used by the final databases.
- Pipeline run metadata exists in the schema and writer path, but the final analysis code does not query or display it, and `run_config` omits prompt version/content, database hash, import command, source JSON hash, and some runtime options such as upstream-depth and prefilter settings.
- `get_git_commit()` may persist `"unknown"` if git metadata is unavailable, so commit provenance is not guaranteed by schema alone.
- Static analysis cannot verify the actual S3 objects, local data files, SQLite contents, file sizes, run rows, or whether the S3 downloads match the databases used to render the committed output files.

## Suggested Follow-up Checks
- Download each documented raw JSON and processed SQLite file, then run `shasum -a 256 <file>` and `wc -c <file>`; record hashes and byte sizes in a checked-in manifest.
- Query each processed DB for provenance: `SELECT * FROM extraction_runs;`, `SELECT DISTINCT source_subreddit FROM users;`, `SELECT MIN(post_date), MAX(post_date), MIN(scraped_at), MAX(scraped_at) FROM posts;`.
- Verify the five downloaded DB filenames are exactly the ones used by `_build_paper_figures.py` and `analysis.ipynb`, then rerun `cd docs/RCT_historical_validation && python _build_paper_figures.py`.
- Add a reproducibility manifest generated during the figure build that includes git commit, input DB paths, DB SHA-256 hashes, selected `extraction_runs.run_id` values, model names, prompt identifiers/content hashes, and build timestamp.
- Recover or regenerate Arctic Shift acquisition metadata for each raw JSON: source URL or API endpoint, exact command/config, subreddit, date range, scrape start/finish date, and raw post/comment input filenames.
- Compare hashes of local `data/*.db` against the S3 processed DBs before accepting any figure/table as traceable to frozen inputs.
