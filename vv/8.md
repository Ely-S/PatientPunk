# V8. Statistical Recalculation

## Verdict
Partial. The final analysis code and generated outputs statically implement the requested responder/non-responder percentages, Wilson 95% intervals, and two-sided exact binomial tests, but V8 is not fully satisfied because the repo does not contain a truly independent recomputation and the README expected p-values are inconsistent with the VV plan/generated outputs.

## Static Evidence
- `docs/RCT_historical_validation/VV_PLAN.md:202-230` defines V8 and expects responder percentage as `positive / n`, non-responder percentage as `(negative + neutral + mixed) / n`, Wilson 95% CIs, two-sided exact binomial tests against `p = 0.50`, responder/non-responder counts summing to `n`, and shared `resp_df` use for Figure 1/Table 3.
- `docs/RCT_historical_validation/_build_paper_figures.py:120-140` implements `_sentiment_breakdown`: `n = len(sentiments)`, `pos` counts only `positive`, `nonr = neg + neu + mix`, `pos_pct = pos/n*100`, `nonr_pct = nonr/n*100`, Wilson intervals via `proportion_confint(..., alpha=0.05, method="wilson")`, and `binomtest(pos, n, 0.5, alternative="two-sided")`.
- `docs/RCT_historical_validation/_build_paper_figures.py:142-165` builds `resp_rows` and `resp_df`; `docs/RCT_historical_validation/_build_paper_figures.py:170-230` uses `resp_df` for Figure 1 bars/error bars, and `docs/RCT_historical_validation/_build_paper_figures.py:297-321` uses the same `resp_df` for Table 3.
- `docs/RCT_historical_validation/_build_paper_figures.py:45-50` and `docs/RCT_historical_validation/requirements.txt:1-10` show the required `scipy` and `statsmodels` dependencies for exact binomial tests and Wilson intervals.
- `docs/RCT_historical_validation/_gen_analysis.py:66-85` duplicates the same `_sentiment_breakdown` logic, and `docs/RCT_historical_validation/_gen_analysis.py:129-133` prints `resp_df[["drug", "n", "pos_pct", "pval", "trial_dir"]]`; this is useful corroboration but not independent because the statistical formulae are copied from the final analysis.
- `docs/RCT_historical_validation/analysis.ipynb:96-104` contains executed output matching the VV plan p-values: famotidine `2.880285e-17`, loratadine `2.063747e-04`, naltrexone `7.905180e-03`, colchicine `1.538599e-01`, paxlovid `6.277709e-01`, prednisone `5.978724e-01`.

## Gaps and Risks
- No separate independent recomputation script/notebook was found. `_gen_analysis.py` and `analysis.ipynb` repeat the same formulas and data extraction pattern as `_build_paper_figures.py`, so they do not satisfy the V8 requirement for independent recalculation.
- `docs/RCT_historical_validation/README.md:123-130` has stale or inconsistent expected p-values for loratadine (`0.0011` vs `2.06e-04`), naltrexone (`0.0082` vs `0.0079`), colchicine (`0.1537` vs `0.1539`), paxlovid (`0.6588` vs `0.6278`), and prednisone (`0.6264` vs `0.5979`). The VV plan, generated code, executed notebook, and HTML output agree with each other, suggesting README drift/data-version drift.
- Static review can confirm formula usage and shared `resp_df` wiring, but cannot verify that the database contents produce the final deduplicated counts without executing queries against the frozen SQLite files.
- Naltrexone is combined through custom logic rather than the generic `best_per_user_drug` path; static inspection shows equivalent ranking/counting intent, but independent recomputation should specifically cover the cross-database user deduplication.
